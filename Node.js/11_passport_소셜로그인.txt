* OAuth 인증 : 개방형 표준 프로토콜 (Open Standard Protocol)로 만들어짐.
  - OAuth가 사용되기 전에는 인증 방식의 표준이 없었기 때문에 기존의 기본 인증인 아이디와 비밀번호를 사용하였는데,
    이는 보안상 취약한 구조임.
  - 기본인증이 아닐 경우는 각 애플리케이션들이 각자의 개발한 회사의 방법대로 사용자를 확인했음.
  > OAuth는 이렇게 제각각인 인증방식을 표준화한 인증방식!
    여러 애플리케이션을 통합해 사용하는 것이 가능해짐.

  - 요청 토큰(request token) : 소비자가 사용자에게 접근 권한을 인증 받기 위해 필요한 정보가 담겨있음.
  - 접근 토큰(access token) : 인증 후에 사용자가 보호된 자원에 접근하기 위한 키를 포함한 값.
    방문증으로 이해할 수 있음. 이 방문증으로 사전에 허락된 공간에 출입할 수 있게 되는 것.

* '페이스북으로 로그인'을 했을 때 진행되는 OAuth 진행 과정.
  1) 사용자는 웹 브라우저에서 로그인 화면을 조회한 후 로그인 요청.
  2) 이 요청을 받으면 패스포트 모듈이 페이스북 서버로 로그인 요청을 함.
  3) 페이스북은 사용자에게 페이스북 로그인 페이지를 보여줌
     # 결국 사용자 인증은 페이스북 서버에서 진행됨!!
  4) 로그인에 성공하면 웹 서버는 그 결과는 알려주는 인증 토큰(Access Token)을 전달 받음.
  5) 페이스북 서버에서 /auth/facebook/callback 패스로 인증 성공 여부와 사용자 정보를 웹 서버로 전달.
  6) 패스포트 모듈이 이 사용자 정보를 데이터베이스에 저장함
  7) 클라이언트에 응답을 보냄.
