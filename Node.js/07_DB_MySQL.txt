* MySQL 데이터베이스 : 관계형 데이터베이스의 일종...
  - 관계형 데이터베이스는 DB 저장소 안에 테이블을 만들어 저장.
  - 어떤 이름의 데이터가 어떤 타입으로 들어갈지를 미리 정의.
  - 한 줄에 해당하는 데이터를 레코드라고 부름.

* RDBMS 다루기
  1) 데이터베이스 연결하기
  2) 테이블 생성하기 : 어떤 칼럼에 어떤 타입의 데이터가 들어갈지 정의.
  3) 레코드 추가
  4) 데이터 조회 : 원하는 조건을 넣어 데이터를 조회함.

* node.js에서 관계형 데이터베이스에 연결할 때는 보통 커넥션 풀을 사용한다.
  - 데이터베이스 연결 객체가 너무 많이 만들어지는 것을 막기 위해 사용!
  - 한 번 연결한 건 다시 사용할 수 있다.
  - 데이터베이스에 연결하면 메모리 리소스를 많이 차지하므로 이렇게 관리함.

  > 중요 < 커넥션 풀은 연결 개수를 제한하므로 연결을 사용한 후에는 반!드!시! 다시 해제해서 풀에 돌려줘야 한다!

* MySQL 데이터베이스에 연결할 때 설정하는 속성들...
  - connectionLimit : 최대 연결 개수 제한
  - host : 호스트 이름. 내 컴퓨터일 경우 localhost나 127.0.0.1
  - port : 포트 이름. MySQL의 디폴트 폴트는 3306
  - user : 데이터베이스 사용자 아이디
  - password : 데이터베이스 사용자의 비밀번호
  - database : 데이터베이스의 이름 설정.
  - debug : 데이터베이스 처리 과정을 로그로 남길 것인지 설정.
