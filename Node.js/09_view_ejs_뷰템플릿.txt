* MVC 패턴이란?
  - Model : 뷰로 표현되는 데이터를 제공하는 것. DB에 데이터를 저장하거나 조회함.
  - View : 눈에 보이는 부분. 사용자에게 결과를 보여주기 위해 만들어진 파일.
  - Controller : 데이터가 처리되는 과정 담당. 사용자 요청을 처리하는 라우팅 함수.
    왜 라우팅 함수를 컨트롤러라고 칠까?
    > 익스프레스에서는 특정 패스로 들어온 요청을 라우팅 함수에서 뿌려주기 때문...

* 뷰 템플릿 : 웹 문서의 기본 형태는 뷰 템플릿으로 만들고 데이터베이스에서 조회한 데이터를 이 템플릿에 적절하게 껴 녛음.
* 뷰 엔진 : 뷰 템플릿을 사용해 결과 웹 문서를 자동으로 생성한 후 응답 전송.
  - 뷰 엔진에는 ejs, pug 등 다양한 종류가 있음.
  - 뷰 템플릿의 종류에 따라 뷰 엔진이 달라짐.

* 뷰 엔진이 동작하는 순서...
  1. 익스프레스 서버 객체인 app의 render() 메소드 호출.
  2. render()가 호출되면 일단 뷰 엔진이 템플릿 파일을 읽어 들임.
  3. 파라미터로 전달한 context 객체의 속성으로 들어 있는 값들을 적용함!
  4. 그 결과를 콜백 함수로 리턴함. 콜백 함수로 전달되는 html 파라미터에는 사용자가 보게 될 html 문서 코드가 들어 있음.
  5. res.end() 메소드를 호출해 이 html 객체를 화면에 뿌려주면 끝!
